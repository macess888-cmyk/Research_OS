# 1. PURPOSE

This experiment calibrates the documentation, observation, and review instruments required to evaluate the Temporal Trust Model.

This experiment does not attempt to prove that TTM is correct.

It asks whether a bounded Research OS environment can reliably record and distinguish the temporal events and trust dimensions defined by TTM.

---

# 2. VALIDATION STAGE

```text
EVP Stage 0 — Pilot Calibration
```

Stage 0 evaluates:

```text
documentation templates
measurement instruments
role clarity
procedural consistency
observation reproducibility
```

No observation produced during this stage constitutes empirical validation of TTM.

---

# 3. EXPERIMENT SUBJECT

The experiment subject is one synthetic operational knowledge unit.

Canonical knowledge unit:

```text
A RuntimeRecordInspectionReport is proposed for use in one bounded
read-only repository review.
```

The knowledge unit is not a production authorization.

It is a controlled research subject.

---

# 4. PRIMARY RESEARCH QUESTION

```text
Can the experiment instruments consistently record and distinguish
the temporal lifecycle and current standing of one bounded knowledge unit?
```

---

# 5. SECONDARY QUESTIONS

```text
Can creation time be distinguished from verification time?
Can verification time be distinguished from execution time?
Can condition change be recorded independently?
Can expiry be distinguished from ordinary ageing?
Can revalidation be distinguished from resetting a timestamp?
Can evidence remain historically intact while current sufficiency changes?
Can authority change independently of evidence?
Can context change independently of scope?
Can two reviewers reproduce the same classification from the same record?
```

---

# 6. TEMPORAL EVENTS UNDER CALIBRATION

The instruments must support separate recording of:

```text
creation_time
verification_time
condition_change_time
expiry_time
execution_time
revalidation_time
```

These events must not be collapsed into one generic timestamp.

---

# 7. TRUST DIMENSIONS UNDER CALIBRATION

The instruments must support separate observation of:

```text
evidence
authority
context
scope
sufficiency
```

No single aggregate trust score is authorized during Stage 0.

---

# 8. CONTROLLED EVENT SEQUENCE

The initial scenario will use the following controlled sequence:

```text
T0 — knowledge unit created
T1 — knowledge unit structurally verified
T2 — operating condition changes
T3 — knowledge unit is proposed for execution-time use
T4 — reviewer determines whether current use remains supported
T5 — revalidation is attempted
T6 — final Stage 0 observation is recorded
```

---

# 9. PREDEFINED SUCCESS CRITERIA

Stage 0 calibration passes only if:

```text
all six temporal events can be recorded separately
all five trust dimensions can be assessed separately
reviewers can identify the same knowledge unit
reviewers can reconstruct the event sequence
reviewers can distinguish historical integrity from current standing
reviewers can distinguish verification from execution-time use
all protocol deviations are visible
all unknowns remain recorded
no result is converted into admission or authority
```

---

# 10. PREDEFINED FAILURE CRITERIA

Stage 0 calibration fails if:

```text
timestamps are ambiguous or collapsed
trust dimensions cannot be separated
reviewers classify the same evidence inconsistently without explanation
historical integrity is mistaken for current sufficiency
prior verification is treated as execution authority
renewal is represented as a simple clock reset
unknown conditions are silently resolved
protocol deviations are omitted
the experiment claims validation of TTM
```

A documented failure remains a valid scientific outcome.
